# Web Starter Audit Pack

## What This Pack Demonstrates

This Tandem Workspace Pack showcases several key capabilities:

1. **Code Auditing**: Systematic review of HTML, CSS, and JavaScript for issues
2. **Accessibility Review**: Identifying WCAG compliance issues and barriers
3. **Quality Assessment**: Evaluating code structure, maintainability, and best practices
4. **Planned Remediation**: Creating structured improvement plans with prioritization
5. **Artifact Generation**: Producing professional HTML audit reports

## Pack Details

- **Estimated Completion Time**: 15-20 minutes
- **Complexity Level**: Beginner-Intermediate
- **Outputs Produced**:
  - Audit findings document (Markdown)
  - Remediation plan (Markdown)
  - Changelog (Markdown)
  - Project Audit Report (HTML)

## Use Cases

- Auditing legacy web projects
- Code review process learning
- Accessibility compliance checking
- Demonstrating Tandem's code analysis capabilities
- Pre-deployment quality gates

## Technical Notes

### File Reading Order

When Tandem processes this pack, files are read in this priority:

1. src/index.html (primary)
2. src/styles.css
3. src/app.js
4. inputs/TODO.md (improvement requirements)

### Audit Categories

The audit covers these categories:

1. **Accessibility**: WCAG compliance, ARIA usage, semantic HTML
2. **User Experience**: Navigation, layout, visual hierarchy
3. **Code Quality**: Structure, maintainability, best practices
4. **JavaScript Functionality**: Bug identification, logic errors
5. **CSS Architecture**: Organization, specificity, responsiveness

### Output Naming Convention

Generated files follow this pattern:

- `outputs/audit_findings.md` - Initial audit results
- `outputs/remediation_plan.md` - Planned fixes
- `outputs/changelog.md` - Changes made
- `outputs/audit_report_YYYYMMDD.html` - Final HTML report
