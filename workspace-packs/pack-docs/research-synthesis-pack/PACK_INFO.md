# Research Synthesis Pack

## What This Pack Demonstrates

This Tandem Workspace Pack showcases several key capabilities:

1. **Multi-Document Synthesis**: Reads and synthesizes information from 10+ Markdown documents simultaneously
2. **Conflict Detection**: Identifies disagreements, conflicting statistics, and contradictory claims across sources
3. **Structured Analysis**: Creates organized tables and frameworks for claims and evidence
4. **Stakeholder Communication**: Adapts technical findings for non-technical audiences
5. **Artifact Generation**: Produces interactive HTML dashboards with data visualization

## Pack Details

- **Estimated Completion Time**: 20-25 minutes
- **Complexity Level**: Intermediate-Advanced
- **Outputs Produced**:
  - Workspace scan summary (Markdown)
  - Synthesis analysis document (Markdown)
  - Claims/evidence table (Markdown)
  - Executive brief (Markdown)
  - Research Brief Dashboard (HTML)

## Use Cases

- Academic literature review and synthesis
- Policy research and briefing document creation
- Competitive analysis across multiple sources
- Technical documentation for executive audiences
- Demonstrating Tandem's synthesis capabilities

## Prerequisites

- Familiarity with research methodology concepts
- Basic understanding of Markdown formatting
- Ability to review and validate synthesized claims
- Access to Tandem workspace environment

## Technical Notes

### File Reading Order

When Tandem processes this pack, files are read in this priority:

1. questions.md (defines research questions)
2. glossary.md (defines terminology)
3. papers/\*.md (research papers, alphabetically)
4. All papers read before synthesis tasks

### Output Naming Convention

Generated files follow this pattern:

- `outputs/workspace_scan.md` - Initial analysis
- `outputs/synthesis_analysis.md` - Common themes and conflicts
- `outputs/claims_evidence_table.md` - Structured claims matrix
- `outputs/executive_brief.md` - Non-technical summary
- `outputs/research_brief_dashboard_YYYYMMDD.html` - HTML dashboard

### Conflict Notation Convention

Disagreements across sources are marked with `[CONFLICT]` and include:

- Source A's claim
- Source B's opposing claim
- Assessment notes
